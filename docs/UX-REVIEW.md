## 4.2 Specific Component Issues

### Issue A-001: Image Gallery Keyboard Navigation
**Location:** `ProjectDetailDialog.tsx`, `ShopifyProducts.tsx`
**Severity:** Medium
**Current:** Images only clickable with mouse
**Fix:**
```tsx
// Add to image grid items
<div
  role="button"
  tabIndex={0}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      handleImageSelect(image)
    }
    if (e.key === 'ArrowRight') focusNextImage()
    if (e.key === 'ArrowLeft') focusPrevImage()
  }}
  aria-label={`Select ${image.name}`}
>
```

### Issue A-002: Skip to Main Content Link
**Location:** `App.tsx` or layout component
**Severity:** High
**Fix:**
```tsx
// Add at top of page layout
<a
  href="#main-content"
  className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:bg-white focus:p-4 focus:rounded focus:shadow-lg"
>
  Skip to main content
</a>

// Add id to main content area
<main id="main-content" className="...">
```

### Issue A-003: HTML Lang Attribute
**Location:** `index.html`
**Severity:** High
**Fix:**
```html
<!DOCTYPE html>
<html lang="en">
```

### Issue A-004: Status Badge Screen Reader Text
**Location:** `VisualProjectCard.tsx`, `VisualJobCard.tsx`
**Severity:** Medium
**Current:** Relies on color + icon
**Fix:**
```tsx
<Badge>
  <StatusIcon className="h-3 w-3 mr-1" aria-hidden="true" />
  <span>{status.label}</span>
  <span className="sr-only"> - {getStatusDescription(status)}</span>
</Badge>
```

### Issue A-005: Form Error Announcements
**Location:** All form components
**Severity:** Medium
**Fix:**
```tsx
// Add to form fields with errors
<Input
  aria-invalid={!!error}
  aria-describedby={error ? `${fieldName}-error` : undefined}
/>
{error && (
  <p id={`${fieldName}-error`} role="alert" className="text-red-600">
    {error.message}
  </p>
)}
```

## 4.3 Screen Reader Testing Notes

Tested with VoiceOver (macOS):

1. **Dashboard** - Reads well, stats cards announce correctly
2. **Projects Page** - Card grid needs explicit list role
3. **Project Wizard** - Step indicator not announced as progress
4. **Shopify Products** - Product count announced, but selection state unclear
5. **Settings** - Token balance reads as plain text, should be live region

### Recommended Live Regions

```tsx
// Token balance should announce changes
<div aria-live="polite" aria-atomic="true">
  <p className="text-4xl font-bold">{tokenAccount?.balance}</p>
  <p className="sr-only">tokens available</p>
</div>

// Processing progress should announce major milestones
<div aria-live="polite" aria-atomic="true">
  {progress === 25 && <span className="sr-only">25% complete</span>}
  {progress === 50 && <span className="sr-only">50% complete</span>}
```
